Fresh Takes Ops Dashboard - PRD

Version 1.0 • Date: April 6, 2026 • Scope: Current Product + Requested Enhancements

Product Summary

Fresh Takes Ops Dashboard is an operational control center for editorial planning, pod performance, production tracking, analytics outcomes, and planner execution for weekly Fresh Take delivery.

Next.js App Router Google Sheets Data Inputs Supabase Persistence Weekly Ops Cadence

1. Goals

  1. Make week-level delivery and throughput performance visible at both POD and writer granularity.
  2. Enable leadership to compare POD throughput and identify top-performing pods quickly.
  3. Preserve all currently working dashboard capabilities while introducing new overview analytics.

3. New Requirement A: Overview Tab KPI Expansion

Objective

Add a leadership overview for last-week output across writer, POD, production throughput, and Gen AI movement.

Required Metrics

  1. Fresh Takes Delivered (Last Week, Approved Beats Only): show counts by writer and by POD.
  2. Throughput per ACD in Production: output count and throughput summary per ACD for the selected week.
  3. Assets moved to Full Gen AI: count of assets transitioned to Full Gen AI in selected week.

UX Requirements

  1. Place metrics in the Overview context with clear POD and writer segmentation.
  2. Add a week filter at top of the section; default remains current existing default behavior.
  3. Allow quick scan cards plus a detailed table below.

Acceptance Criteria

  1. When week changes, all three overview metrics refresh in one pass.
  2. Writer and POD counts reconcile with the same approved-beat source for that week.
  3. Empty state appears when no approved beats exist for selected week.

4. New Requirement B: Editorial Throughput Benchmarking

Objective

In Editorial, provide a pod-level first view and writer-level second view focused on throughput so we can identify the best POD by throughput.

Required Views

  1. POD Level Throughput View (Default): rank PODs by throughput and delivery efficiency.
  2. Writer Level Throughput View: drill down into individual contributor throughput within pod context.

Suggested Throughput Fields

  1. Approved beats delivered
  2. Scripts completed
  3. Production handoff count
  4. Median turnaround (optional if available)
  5. Throughput score (normalized metric for ranking)

Acceptance Criteria

  1. POD view loads first and supports rank ordering by throughput.
  2. Writer view is filterable by week and POD.
  3. “Best POD” indicator is clearly displayed with metric basis.

5. Data and Logic Definitions

Metric Definition Source
Approved Beat Beat row that satisfies approved status logic in tracker source for selected week. Google Sheets + dashboard normalization layer
Writer Delivery Count Unique approved beats delivered by writer in selected week. Editorial/overview API payloads
POD Delivery Count Sum of unique approved beats delivered by writers mapped to that POD. Writer-to-POD mapping + editorial payloads
ACD Throughput Assets or beats processed in production by ACD during selected week. Production dataset / ACD metrics endpoint
Full Gen AI Count Number of assets marked as moved to Full Gen AI in selected week. Production + classification tags

6. Non-Functional Requirements

  1. Keep dashboard responsive on desktop and mobile; no horizontal overflow on core tables without explicit scroll wrappers.
  2. Maintain existing theme toggle behavior (light/dark) and sticky header UX.
  3. API-backed sections should fail gracefully with warning notes, not blank screens.

7. Out of Scope (For This PRD Iteration)

  1. Changing planner data model or planner interaction patterns.
  2. Redesigning analytics next-step logic.
  3. Historic backfill corrections beyond current weekly data windows.

8. Open Questions
